Critical Threat

Supply Chain Incident: Malicious Dependency Poisoning

24h AGO

1. How the Attack Was Pulled Off

Attackers compromised a popular open-source utility package by registering a typo-squatted domain and gaining maintainer credentials through credential stuffing. They injected an obfuscated loader script into the build pipeline that exfiltrated environment variables and API keys during `npm install` execution.

2. Attack Vector Analysis

  • Vector: Typosquatting / Credential Harvesting
  • Impact: Remote Code Execution (RCE) on CI/CD Runners
  • Payload: Base64-encoded reverse shell via NodeJS child process

3. How to Defend & Mitigate

# Lock down package dependencies using lockfile integrity checks
npm ci --ignore-scripts
# Scan project for vulnerable or malicious packages
npm audit --audit-level=high
Free Breakdown

Active Directory Pass-the-Hash Exploit

3 DAYS AGO

1. How the Attack Was Pulled Off

After acquiring local admin access via a phishing payload, attackers used Mimikatz to dump NTLM password hashes directly from the LSASS memory space. Using the extracted hash, they authenticated to adjacent domain controllers via SMB without needing the plaintext password.

2. How to Defend & Mitigate

  • Enable LSA Protection: Prevent non-system processes from inspecting LSASS memory.
  • Credential Guard: Enable Virtualization-based Security (VBS) on Windows endpoints.
  • Implement PAM: Restrict local administrator password reuse using Microsoft LAPS.
